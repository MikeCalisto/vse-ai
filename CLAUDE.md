# Сайт AiSashka — инструкции для Claude Code

Этот файл читается автоматически в начале каждой сессии. Всегда следуй правилам ниже.

## Что это

Это git-репозиторий **vse-ai** (`https://github.com/MikeCalisto/vse-ai`), задеплоенный на Vercel: **`https://vse-ai.vercel.app`**. Главная ветка — `main`. После каждого `git push` Vercel автоматически передеплоивает.

Структура «один сайт — много подстраниц». Каждая подстраница живёт в своей подпапке. Сейчас работают:

| URL | Папка | Описание |
|---|---|---|
| `vse-ai.vercel.app/` | `zakrytyi-klub/` | Главный лендинг — закрытая Telegram-спільнота VSE AI. |
| `vse-ai.vercel.app/product-ai-content/` | `product-ai-content/` | Лендинг курса «Предметна зйомка через АІ». |

Корень репозитория содержит только **общие** файлы:
- `vercel.json` — конфиг Vercel (rewrites, redirects, headers)
- `.gitignore`
- `CLAUDE.md` (этот файл)
- подпапки подстраниц

## ⛔ ГЛАВНОЕ ПРАВИЛО

**Когда добавляешь новую подстраницу — не редактируй и не удаляй файлы существующих рабочих подстраниц.** Существующие лендинги уже залиты в продакшн и приносят трафик. Правки в чужой подпапке = риск сломать рабочий сайт.

Допустимы правки в чужой подпапке **только** если пользователь явно об этом попросил («поменяй то-то на закрытом клубе», «обнови оферту на product-ai-content» и т.п.).

## Правила работы

### Создание новой подстраницы

1. Создай новую папку в корне, например `<slug>/`. Slug — короткое латинское имя (`new-course`, `webinar`, `freebie`).
2. Положи внутрь:
   - `index.html` — сама страница
   - все её картинки/видео/шрифты — **только в этой папке**, чтобы не пересекаться с другими подстраницами
   - если нужна оферта/политика — отдельные `oferta.html`, `politika.html` **внутри той же папки** (чтобы кнопка «Повернутися» вела на эту же подстраницу)
3. Внутри HTML используй **относительные** пути к ассетам (`<img src="photo.jpg">`, не `/photo.jpg`).
4. URL подстраницы — `vse-ai.vercel.app/<slug>/`. Чтобы это работало без `/index.html` в адресе, добавь в `vercel.json` редирект:
   ```json
   { "source": "/<slug>", "destination": "/<slug>/index.html", "permanent": false },
   { "source": "/<slug>/", "destination": "/<slug>/index.html", "permanent": false }
   ```
5. Также **исключи** новый slug из catch-all rewrite `/((?!product-ai-content|zakrytyi-klub).*)` — добавь его в негативный лист:
   ```json
   { "source": "/((?!product-ai-content|zakrytyi-klub|<slug>).*)", "destination": "/zakrytyi-klub/$1" }
   ```
   Иначе все ассеты новой подстраницы попадут под rewrite и провалятся в `zakrytyi-klub/`.
6. Желательно — `Cache-Control: max-age=0, must-revalidate` для подстраницы (см. `vercel.json` → `headers`).

### Картинки и вес

Большие фото (>1 MB) перед коммитом всегда сжимать. На macOS быстрый способ:
```bash
sips -s format jpeg -s formatOptions 82 -Z 1400 input.png --out output.jpg
```
Цель: каждая картинка ≤ 300 KB.

### Деплой и кэш

- `git push` в `main` → Vercel сам деплоит за ~30 сек.
- Если пользователь видит старую версию после деплоя — это кэш браузера (особенно Telegram in-app, мобильный Safari). Заголовки `must-revalidate` стоят, но мобильные браузеры могут игнорировать.
- Лучший workaround при кэш-проблемах: открывать с `?v=<число>` или сделать redirect `/path/` → `/path/index.html`.

### Что нельзя без явной просьбы

- Менять `vercel.json` (кроме добавления конфига для новой подстраницы).
- Трогать `.git/`, переписывать историю, делать `--force push`.
- Удалять файлы существующих подстраниц.
- Менять домен или origin.
- Делать редиректы со старых URL без подтверждения.

## Текущий `vercel.json` (что в нём есть)

- **Rewrites:** все запросы на корневые пути (кроме `/product-ai-content/*` и `/zakrytyi-klub/*`) внутренне отдают `/zakrytyi-klub/...`. Так URL `vse-ai.vercel.app/` показывает закрытый клуб без необходимости таскать его файлы в корень.
- **Redirects:** `/product-ai-content` и `/product-ai-content/` редиректят на `/product-ai-content/index.html` (обход кэша браузера).
- **Headers:** `/product-ai-content/*` идёт с `Cache-Control: max-age=0, must-revalidate`.

## Полезное

- Локальный preview: `npx serve -l 8000` из корня репо.
- Проверка деплоя: `curl -sI https://vse-ai.vercel.app/<путь>` — должен возвращать 200 (или 307 для редиректов).
- Cмотри последние коммиты в `git log --oneline -20`, чтобы понять контекст недавних изменений.
